# Ручная проверка: несколько одинаковых станков

Сценарий: в группе **ЧПУ** по **два** станка одного типа — фрезерные `ФРЕЗ-ЧПУ-01` / `ФРЕЗ-ЧПУ-02` и токарные `ТОКАР-ЧПУ-02` / `ТОКАР-ЧПУ-03`. Заказ с большим количеством штук должен **параллельно** загрузить оба станка одной операции.

## Подготовка

1. Установите `jq` (`brew install jq`).
2. Из корня репозитория:

```bash
chmod +x scripts/prep-duplicate-machines-demo.sh
./scripts/prep-duplicate-machines-demo.sh
```

Скрипт скопирует `data/schedule-duplicate-machines.example.json` → `data/schedule.json`.

3. Запустите (или перезапустите) сервер:

```bash
mvn -q exec:java
```

4. Загрузите тестовый заказ:

```bash
./scripts/prep-duplicate-machines-demo.sh --no-reset
```

По умолчанию: заказ **`З-дубль-станки`**, по **12** шт. `вал-буровой` и `корпус-бура`.  
В шаблоне `schedule-duplicate-machines.example.json` задано `"scheduling": { "overlapBatches": true }` — overlap черновая/чистовая и фреза/расточка.  
Параметры: `DEMO_PART_QTY=16 DEMO_ORDER_ID=З-тест ./scripts/prep-duplicate-machines-demo.sh --no-reset`

5. Откройте: [http://localhost:8080/schedule?format=html](http://localhost:8080/schedule?format=html)

## Что смотреть на экране

### Парк станков

- Блок **«Парк станков»** / обзор цеха: **8** станков (было 6 + 2 дубля ЧПУ).
- В группе **ЧПУ** четыре строки: две фрезы, два токаря.

### Расписание по станкам

Прокрутите к **«Расписание по станкам»**, пресет масштаба **«Неделя»** или **«Смена»**.

| Станок | Ожидание |
|--------|----------|
| `ТОКАР-ЧПУ-02` и `ТОКАР-ЧПУ-03` | У обоих есть полосы **черновая токарка** по заказу `З-дубль-станки` (вал). Штуки **разделены** между станками (не все 12 только на одном). |
| `ФРЕЗ-ЧПУ-01` и `ФРЕЗ-ЧПУ-02` | У обоих **черновая фрезеровка** (корпус). Аналогично — нагрузка на двух фрезах. |
| `РАСТОЧ-03`, `ШЛИФ-04` | Работа по корпусу/валу после черновых операций (один станок на тип — как в базовом демо). |

Фильтр: отметьте заказ **`З-дубль-станки`** — останутся только его операции.

### Пакет и переналадка

- На **каждом** токарном в начале пакета черновой — **одна** оранжевая переналадка, дальше подряд штуки без переналадки между ними.
- То же на фрезерных для черновой фрезеровки.

### Параллельность по времени

- На двух токарных полосы **черновой** должны **перекрываться по времени** (оба станка работают в одни смены, а не строго друг за другом).
- **Чистовая на `ТОКАР-ЧПУ-03`** должна **начаться раньше**, чем закончится последняя черновая на `ТОКАР-ЧПУ-02` (overlap при двух токарных станках).
- Сравните с одним токаром: при `DEMO_PART_QTY=4` планировщик может занять только один станок — для явного эффекта дублей держите **≥ 10** шт.

## Проверка через API (опционально)

```bash
curl -s http://localhost:8080/schedule | jq '
  [.orders[] | select(.orderId=="З-дубль-станки") | .parts[] |
    select(.partId=="вал-буровой") | .assignments[] |
    select(.taskId=="черновая-токарка" and .status!="CANCELLED") |
    .machineId] | group_by(.) | map({machine: .[0], count: length})'
```

Ожидание: **два** объекта с разными `machineId` (`ТОКАР-ЧПУ-02` и `ТОКАР-ЧПУ-03`), сумма `count` = 12.

Аналогично для фрезы:

```bash
curl -s http://localhost:8080/schedule | jq '
  [.orders[] | select(.orderId=="З-дубль-станки") | .parts[] |
    select(.partId=="корпус-бура") | .assignments[] |
    select(.taskId=="черновая-фрезеровка" and .status!="CANCELLED") |
    .machineId] | group_by(.) | map({machine: .[0], count: length})'
```

## Дополнительно: смена и переплан

1. Сдвиньте время до конца смены:

```bash
curl -s -X PUT http://localhost:8080/time \
  -H 'Content-Type: application/json' \
  -d '{"currentTime":"2026-05-22T18:00:00Z"}'
```

2. В HTML внизу — **«Закрытие смены»**: введите факты по **всем** строкам (на обоих токарных и обеих фрезах — свои числа), **«Закрыть все смены и перепланировать»**.

3. Убедитесь:
   - в **прошлой** смене остались только фактически выполненные операции (без «хвоста» отменённого плана);
   - на **следующей** смене хвост снова может идти на **оба** станка того же типа, без переналадки перед каждой штукой пакета.
