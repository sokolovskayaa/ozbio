# Ручное тестирование и согласование с директором

Пошаговый сценарий встречи для MVP планировщика. Эталон поведения: [правила-планировщика.md](правила-планировщика.md).

| Параметр | Значение |
|----------|----------|
| Длительность | 45–60 мин |
| Экран | **http://localhost:8080/schedule?format=html** |
| Запуск | `mvn spring-boot:run` |

**Подготовка:** `./scripts/setup-local-postgres.sh` → `mvn spring-boot:run` (или Docker: `docker compose up -d` + профиль `docker`)

## Подготовка оператора (до встречи)

- [ ] `./scripts/setup-local-postgres.sh` (если ещё не делали) → `mvn spring-boot:run`
- [ ] `./scripts/prep-director-demo.sh` без ошибок
- [ ] В браузере открыт http://localhost:8080/schedule?format=html
- [ ] На втором мониторе: [правила-планировщика.md](правила-планировщика.md) (§4–§6)

### Вступление для директора (~30 сек)

- Это **прогнозный план** на основе справочника технологий и парка станков.
- **Готовность заказа** — когда закончится **весь** заказ (самая долгая деталь).
- План **круглосуточный** (без смен и ночных окон в этой версии).
- На одном станке: сначала **все** штуки черновой, затем чистовая (классический пакет).

---

## Сценарий 1. Первый заказ: две детали, пакет по операциям

**Цель:** «сначала все штуки — операция 1, потом все — операция 2» и параллельность вал/корпус.

### Действия

1. Кнопка **«+ Добавить заказ»** или форма в «Управление планом».
2. Детали: **вал-буровой 8 шт.** + **корпус-бура 8 шт.** (номер можно не указывать — `З-2026-…`).
3. Таблица заказа: **готовность**, **резерв** (у корпуса резерв меньше).
4. Операции: у вала 8× черновая токарка → чистовая → шлифование; у корпуса — фрезеровка → расточка → чистовая.

### Ожидание

| № | Ожидание |
|---|----------|
| 1.1 | План на **все 8+8** штук |
| 1.2 | Все **8** черновых токарок **до** первой чистовой |
| 1.3 | Вал и корпус **параллельно** на разных станках |
| 1.4 | Готовность заказа ≈ конец цепочки **корпуса** (приоритет 10) |

**Вопрос:** «Сначала весь черновой проход партии на станке — так в цехе?»

|--------|------------|
| ☐ | ☐ Да ☐ Нет ☐ Частично |

## Сценарий 2. Второй заказ в очереди

**Цель:** новый заказ не ухудшает срок первого.

### Действия

1. Запомнить **readyAt** первого заказа: ____________________
2. Добавить заказ: **муфта-зажимная 1 шт.**
3. Снова **readyAt** первого: ____________________

### Ожидание

| № | Ожидание |

| 2.1 | Муфта короче, планируется **после** первого заказа в очереди |
| 2.2 | **readyAt** первого **не позже** чем до второго заказа |

**Вопрос:** «Новый заказ не отодвигает дату старым клиентам?»

## Сценарий 3. Расписание по станкам

**Цель:** переналадка, загрузка, фильтры.

### Действия

1. Блок **«Расписание по станкам»**.
2. Пресеты **День / 3 дня / Неделя**.
3. **ТОКАР-ЧПУ-02**: черновая токарка → **оранжевая переналадка** → чистовая.
4. **ФРЕЗ-ЧПУ-01** и **РАСТОЧ-03**: корпус на разных станках (расточка **после** начала фрезеровки по технологии, но может **пересекаться** по времени с фрезеровкой других штук).
5. Фильтр по заказу и детали.
6. Статусы **Занят / Свободен** в парке станков.

### Ожидание

| № | Ожидание |
|---|----------|
| 3.1 | Шкала непрерывная (24/7), без «серых» ночных зон смен |
| 3.2 | Переналадка перед первой операцией и между типами (черновая → чистовая) |
| 3.3 | Подписи по-русски |

**Вопрос:** «Так удобно смотреть загрузку на неделю?»

## Сценарий 4. Справочник и ограничения MVP

### Справочник деталей

- Корпус **10**, вал **8**, муфта **3** — приоритет **внутри заказа**.

### Ограничения (что сказать директору)

| Тема | Статус в MVP |
|------|----------------|
| Смены 08–20, закрытие смены | **Не в MVP** — следующая фаза |
| Симуляция времени | **Не в MVP** — «сейчас» = реальное время |
| Поломка станка (DOWN) через API | **Не в MVP** |
| Два токарных / overlap чистовой | **Не в MVP** |
| Склейка партий из разных заказов | **Не в MVP** |

**Вопрос:** «Что критично для пилота, что — следующая фаза?»

## Команды

```bash
./scripts/setup-local-postgres.sh   # один раз, если role "ozbio" does not exist
mvn spring-boot:run
./scripts/prep-director-demo.sh
open 'http://localhost:8080/schedule?format=html'
```

Добавить заказ через API:

```bash
curl -s -X POST http://localhost:8080/orders \
  -H 'Content-Type: application/json; charset=utf-8' \
  -d '{"orderId":"З-2026-0142","parts":[{"partId":"вал-буровой","quantity":8},{"partId":"корпус-бура","quantity":8}]}'
```
