# Тестирование MVP планировщика

Зафиксированная версия (май 2026): пакетное планирование (overlap **по умолчанию выкл.**, `scheduling.overlapBatches`), overlap на дублях станков при вкл. флаге, закрытие всех смен с одним перепланом, скрытие отменённых в отчёте.

## Запуск

```bash
mvn -q test
```

Без Maven (как в CI-скриптах):

```bash
find src/main/java -name '*.java' | xargs javac -d target/classes -cp lib/gson-2.11.0.jar
find src/test/java -name '*.java' | xargs javac -d target/test-classes \
  -cp "target/classes:lib/gson-2.11.0.jar:lib/junit-jupiter-api-5.11.4.jar"
java -jar lib/junit-platform-console-standalone-1.11.4.jar \
  -cp "target/classes:target/test-classes:lib/gson-2.11.0.jar" --scan-class-path
```

## Добавление теста

При изменении логики в `src/main/java/scheduler/`:

1. Добавьте или обновите unit/integration тест в `src/test/java/scheduler/`.
2. Обновите эту матрицу и при необходимости [правила-планировщика.md](правила-планировщика.md).
3. Запустите `mvn -q test`.
